如果要在Linux系统跑Windows程式，只要能拿到exe，就有机率能透过"Wine"转译技术执行。

它能够把Windows应用程式变成有如原生Linux程式一样，让使用者以接近原生的效能，操作Windows上的软体。

图例：Linux系统执行Windows版的WinRAR

本文Ivon详细介绍Linux系统使用Wine跑Windows软体的方法。首先简介Wine的由来，再介绍Wine指令用法，最后附上一些额外的Wine辅助工具。

1. Wine是什么
#

Wine (WineHQ，红酒) 是从1993年发展至今的exe转译软体，它將Windows exe的指令即时转译成Linux的系统呼叫，连3D绘图指令都一併转译，让使用者能够在类Unix系统（Linux、FreeBSD、macOS）执行Windows程式。

Wine为自由开源软体。名字原本为"Wine Is Not an Emulator"的缩写，代表它不是虚擬机也不是模擬器，而是转译器，性能损失很小。

在Wine的加持下，诸如Microsoft Office 2016、Adobe Photoshop CC 2021、Adobe AutoCAD、侠盗猎车手V等程式，都能用Wine在Linux执行，甚至有人回报说Linux跑游戏比Windows还顺畅。

Wine具体能执行的Windows程式有哪些呢？建议到 WineDB和 ProtonDB看看，里面有收录各个Windows软体的运作状况，还有安装前的小撇步。

Steam Deck也有採用Wine的技术，叫做Proton。因此有上Steam的Windows游戏用 Linux版Steam客户端会更容易启动，这些游戏已经由许多Linux系统与Steam Deck玩家测试过了。

除了官方版的WineHQ以外，Wine还有很多分支(fork)，例如Steam Proton就是专为游戏优化的，还有外国玩家维护的GE-Proton、中国人维护的Deepin-Wine等版本。通常使用官方版Wine就好。

2. 给Linux Wine新手的忠告
#

这边我列出三点最重要的事情，免得使用者对Wine有错误的期待。

Wine並非万能
#

因为Windows API黑箱的特性，软体能不能运作要看运气。

Wine不是虚擬机软体，无法模擬Windows系统的全部功能，別妄想任何软体都能用Wine解决。

如果你的软体需要用到外接硬体设备，得安装驱动程式，那么高机率跑不起来，比如绘图板、印表机。此外需要虚擬化、CUDA、Windows Store的UWP软体也不行用Wine跑。

需要完整Windows环境又不想安装双系统的，请考虑安装 Windows QEMU/KVM虚擬机

或者寻找Linux系统上同类软体替代品： Linux电脑必装软体

使用Wine照样有中毒风险
#

甭多说了。Windows指令一翻译成Linux，勒索病毒照样可以加密你硬碟的档案。

这不是说Linux不安全，而是Windows程式的指令经由Wine翻译成Linux之后就跟系统原生程式没什么两样了，要搞破坏照样拦不住。

请不要从来路不明的网站下载exe。

请优先尝试图形化界面
#

Wine预设是要用指令操作的，这篇文章会使用大量Wine的指令。儘管对Linux用户来说打指令並非难事，不过如果只是想简单装个exe软体，用別人开发好的图形前端操作会更简单，他们可以大大简化Wine安装软体的痛楚。

通用exe我会使用 Bottles for Linux

装游戏的话可以用 Steam Proton或 Lutris

3. 如何安装Wine
#

你需要一台x86架构的Linux电脑。因为大部分Windows exe都是x86 (32位元) 或x64 (64位元) 架构的，Wine目前仅支援x86架构的Linux系统。ARM架构的Linux系统（如Raspberry Pi开发板）支援状况不佳，且会涉及到异构计算的问题。

这里我提供二种Wine的安装方式，择一安装。

原生套件版
#

Ubuntu、Debian、Arch Linux、Manjaro、Fedora、Gentoo、openSUSE等常见的发行版都有收录Wine套件。

  1. 以Ubuntu系统为例，虽然官方套件库有收Wine套件，但是版本陈旧。 Wine官方文件建议加开发者的套件库，取得最新版Wine。

  2. 给AMD64架构的Ubuntu系统启用i386支援

sudo dpkg --add-architecture i386
  1. 新增WineHQ的套件库
# 取得WineHQ的金钥
sudo mkdir -pm755 /etc/apt/keyrings

sudo wget -O /etc/apt/keyrings/winehq-archive.key https://dl.winehq.org/wine-builds/winehq.key

# 加入WineHQ套件库。注意os变数后面的版本代號，Ubuntu 22.04是为jammy
os=jammy
sudo wget -NP /etc/apt/sources.list.d/ https://dl.winehq.org/wine-builds/ubuntu/dists/$os/winehq-$os.sources

# 更新套件库
sudo apt update
  1. 接著安装稳定版的Wine。
sudo apt install --install-recommends winehq-stable

Flatpak版
#

Flatpak版可以透过容器机制，將Wine与主系统隔离。因为Flatpak跨发行版，比较不会遇到系统套件依赖不全的问题。

Flatpak版的Wine內建Winetricks和DXVK。

  1. 安装 Flatpak，之后再从Flathub安装Wine。
flatpak install flathub org.winehq.Wine
  1. Flatpak的Wine启动指令都要加上flatpak run，因此flatpak run org.winehq.Wine等於wine指令。
flatpak run org.winehq.Wine
  1. Flatpak版所有的WINEPREFIX都会储存在~/.var/app/org.winehq.Wine/data

4. Wine指令基本用法
#

详细內容请参考官方文件 Wine User’s Guide

  1. Wine的用法：
wine "exe路径"

  1. 我们以简单的Windows软体「Notepad++」当例子。

  2. 如果你要执行notepad++.exe或notepad++.msi，那么使用以下指令就可以启动程式：

wine notepad++.exe

wine notepad++.msi

  1. 等一会画面出来就可以使用了。Notepad++执行状况良好。

  2. 如果Windows软体需要安装，那么Wine会在安装好后帮你建立桌面捷径。

  3. 另外Wine会自动註册档案关联，你在Linux档案管理器对exe点两下，Wine通常会自己打开。若不想要Wine自行关联Linux程式档案，请用winecfg指令关闭Manage File Associations。

  4. 还请注意不是每个exe都能这样直接开，有的需要你安装额外的Windows套件才可以开启。比如有许多软体依赖.Net Framework、Visual C++ Redistributable、DirectX等等，或是需要用到中文字体，这些Wine没有內建，你得用Winetricks安装（后述）。

  5. 你可能会问，Wine把Windows软体安装到哪里去呢？请看下一节的WINEPREFIX概念。

5. 理解WINEPREFIX虚擬C槽的概念
#

WINEPREFIX指的是Wine存放资料的目录。

Wine初次启动的时候，会在使用者家目录建立~/.wine目录存放资料，里面的结构很像Windows的C槽，含有登录档、常用系统文件、字体等等。使用者的exe也是安装到这里。

WINEPREFIX可以理解为虚擬C槽的概念吧！如果你用wine explorer指令开启Wine的档案总管，就会看到(C:)槽和(Z:)槽两个磁碟，(C:)对应的路径是~/.wine，而(Z:)即对应整个Linux系统的根目录了。

你可以使用WINEPREFIX环境变数，后面接路径，用於告诉Wine资料存在哪里。

用法是这样，在wine指令前面加上环境变数，这样Wine就会改成在~/.wine32建立虚擬C槽了。如果不写WINEPREFIX，Wine资料预设储存路径就是~/.wine。

WINEPREFIX="~/.wine32" wine notepad++.exe

此外，wine还有一个叫做WINEARCH的环境变数，指定Wine环境是32位元还是64位元。你应该知道有的程式真的没办法在64位元执行！

WINEARCH=win32 wine <exe>

预设情况下，Wine建立的环境都是64位元环境。你必须明確指定WINEARCH才可以用32位元跑。

不过除非是很老的程式，否则64位元的Wine环境跑32位元exe大部分时候是没什么问题的，不太需要指定建立纯32位元环境。

那么WINEPREFIX有何用处呢？它常常用来建立多个不同的「Wine容器」，类似多个虚擬C槽的概念。有些很机车的Windows软体，需要改一堆登录档，甚至把架构改32位元才能成功执行，这样久了Wine的C槽环境不免会乱掉。所以，利用WINEPREFIX建立多重的Wine容器，將不同用途的exe隔离就是个解决办法。

比如，我可以用这些指令在多个Wine的容器之间切换：

# 专门跑32位元程式的容器
WINEPREFIX="/home/user/.wine32" WINEARCH=win32 wine notepad++.exe

# 专门跑Photoshop 2021的容器
WINEPREFIX="/home/user/.winephotoshop" wine photoshop.exe

6. Wine內建的辅助工具
#

Wine本体模擬了一系列Windows系统的程式，输入指令开启。

虽说是图形程式，但他们也可以用指令操作来变更Windows系统设定，方便用在指令稿批次处理。详情请参考 Wine Wiki

explorer：模擬档案总管。启动指令：wine explorer，下面程式以此类推。

control：模擬控制台，仅能解除安装程式和调整网路设定

taskmgr：模擬工作管理员。值得一提的是Windows exe的行程在Linux端用htop指令也看得到。

regedit：模擬登录档编辑器

notepad：模擬记事本

cmd：命令提示字元，支援DOS指令。

wine-dbg：除错工具

winecfg：调整Wine设定值。这个比较有用所以我特別讲一下winecfg的功能。

应用程式 → 调整模擬的Windows系统版本

显示 → 萤幕解析度，调整Windows程式的字体大小(DPI)

显示 → 模擬一个虚擬桌面，可让Windows程式跑在一个虚擬桌面的视窗。这样你可以防止某些程式强制进入全萤幕。

桌面整合 → MIME类型，可关闭Wine与Linux系统的档案关联机制。

函式库 → 这里设定DLL覆写，设定要使用Wine提供的DLL还是Windows程式提供的DLL。

7. 用Winetricks简化安装中文字体流程
#

Winetricks是一个指令稿，用於帮使用者快速安装特定Windows软体。

比如装中文字体、修改登录档、建立容器等。有些软体安装后还要修改登录档才能被系统认到，Winetricks可以將这些操作自动化。

  1. 安装方法：从 Winetricks的Github下载最新版，並將其移动到系统目录。
wget https://raw.githubusercontent.com/Winetricks/winetricks/master/src/winetricks
chmod +x winetricks
sudo mv winetricks /usr/bin

  1. Flatpak版Wine已內建Winetricks，启动指令为：flatpak run --comamnd=winetricks org.winehq.Wine

  2. Winetricks的使用方法：输入winetricks指令开启图形界面，按照画面指示操作。

或者你可以直接输入Windows套件名称，叫Winetricks自动下载安装。这里的例子是告诉Winetricks，下载中文字体cjkfonts和dotnet471这二个套件並安装。

winetricks cjkfonts dotnet471

结合我们上面讲过的WINEPREFIX概念，你可以用该变数告诉Winetricks要变更哪一个Wine的容器。

WINEPREFIX=~/.wine winetricks cjkfonts dotnet471

8. 安装DXVK强化3D效能
#

DXVK是把DirectX绘图指令转译成Vulkan的程式，可以有效增进游戏相容性，並提昇游戏效能。

它还可以测定应用程式的GPU资讯、显示FPS。

  1. 如何安装DXVK：到 Github Releases下载最新版zip档，解压缩

  2. 接著將DLL档复製到Wine的安装目录（预设应为~/.wine）

export WINEPREFIX=~/.wine

cp x64/*.dll $WINEPREFIX/drive_c/windows/system32

cp x32/*.dll $WINEPREFIX/drive_c/windows/syswow64
  1. 启动winecfg
winecfg

  1. 到函式库页面，设定DLL覆写，將d3d11、d3d10core、dxgi、d3d9调整为「內建先於原生」。

  2. 如何確认DXVK安装成功：使用环境变数DXVK_HUD执行Windows 3D应用程式，看看HUD有无跑出来就知道了。

DXVK_HUD=1 wine game.exe
